Remotion Content Assets Integration Plan

Plan to add a deterministic, lower-cost content asset lane to the b2c-mobile-business-launch skill for cases where Higgsfield is not approved, not desired, or not the best tool for the job.

Prepared: 2026-05-27 Repo: b2c-mobile-business-launch-skill Status: plan only

Decision

Add Remotion as a first-class content-production route, not merely a weak fallback. Higgsfield remains the premium generative path for mascots, AI visuals, presenter-style ads, image-to-video, and photoreal concepting. Remotion becomes the preferred path for assets that should be code-generated from product truth: screenshots, UI recordings, brand tokens, copy variants, data, and repeatable campaign formats.

Why Now

The current skill already warns against silently downgrading Higgsfield to hand-authored assets. The missing piece is a credible local production system that future agents can scaffold, render, audit, and rerun. Remotion fits because it uses React, CSS, SVG, Canvas, WebGL, code, and reusable composition primitives to create real video and image outputs.

Use Higgsfield

  • Net-new AI images, mascots, app icons, and style exploration.
  • Presenter, avatar, image-to-video, or UGC ad generation.
  • Photoreal lifestyle or campaign visuals where source UI is insufficient.
  • Virality prediction or marketing-studio workflows are explicitly desired.

Use Remotion

  • App preview clips, screen-recording edits, and product demo loops.
  • Store screenshots, social posts, ad variants, and carousels from real UI.
  • Founder-led TikTok/Reels/Shorts overlays, captions, and hook variants.
  • Batch-rendered assets tied to DESIGN.md tokens and copy data.

Do Not Use Either

  • No truthful product UI, owned media, or licensed public asset source exists.
  • The asset would imply unsupported app behavior, price, endorsement, or result.
  • The target is public posting or paid spend without founder approval.
  • Remotion license eligibility is unclear for a larger commercial entity.

Implementation Phases

1. Add The Skill Reference

Create skill/b2c-mobile-business-launch/references/remotion-content-assets.md. It should define triggers, source inputs, license routing, setup, composition standards, render commands, QA checks, output storage, and founder-only gates.

It should also include a compact map to the upstream Remotion skill rules: compositions.md, assets.md, parameters.md, timing.md, text-animations.md, measuring-text.md, images.md, videos.md, audio.md, subtitles.md, and ffmpeg.md.

2. Wire The Reference Into Existing Routes

  • Update SKILL.md body items for visual system, store ops, UGC/Fastlane, tool recipes, and deliverable standards.
  • Avoid frontmatter bloat unless necessary; the description has a validator-enforced character ceiling.
  • Update paid-tool-routing.md so Higgsfield's fallback column names Remotion after founder approval and labels Remotion as license-check-required, not always free.
  • Update tool-recipes.md with a new Remotion section next to Higgsfield, including docs-refresh requirements and CLI examples.
  • Update ugc-creator-engine.md, app-store-listing-prep.md, onboarding-conversion.md, and artifact-contracts.md where content assets already appear.

3. Add A Content Asset Contract

Add CONTENT_ASSETS.md, content-assets.html, and a small manifest contract to the artifact map. The contract should make assets inspectable before launch work depends on them. 

{
  "asset_id": "tiktok-hook-01",
  "surface": "tiktok_reels_shorts",
  "route": "remotion",
  "composition_id": "VerticalHookDemo",
  "dimensions": "1080x1920",
  "duration_seconds": 12,
  "inputs": ["screenshots/raw/onboarding.png", "DESIGN.md", "copy/hooks.json"],
  "outputs": ["content-assets/out/tiktok-hook-01.mp4"],
  "truth_constraints": ["real app UI visible", "no unsupported claims"],
  "render_proof": "npm run render:content -- --id tiktok-hook-01",
  "status": "draft"
}

4. Decide The Remotion Project Shape

Do not bundle Remotion dependencies inside this skill. Instead, have the skill scaffold a launch-repo-local Remotion workspace only when a launch needs content rendering. 

content-assets/
  CONTENT_ASSETS.md
  content-assets.html
  manifest.json
  copy/
  inputs/
  remotion/
    package.json
    remotion.config.ts
    src/
    public/
  out/

This keeps the skill lightweight while making rendered assets live next to the launch package they belong to.

5. Add Templates Without Vendoring The Whole Upstream

Add templates for CONTENT_ASSETS.md, content-assets.html, manifest.json, and a minimal Remotion starter README. Do not copy the entire remotion-dev/skills repo into this skill. Instead, reference the installed remotion-best-practices skill when available and require a docs refresh when generating Remotion code.

6. Add Validator And LaunchBench Coverage

  • Add scripts/check-content-assets.ts to inspect route, fallback approval, license notes, manifest outputs, render proof, source inputs, and public-claim constraints.
  • Add the validator to package.json, run-launchbench.ts, and run-validator-fixtures.ts.
  • Add LaunchBench scenarios: higgsfield-remotion-fallback-unapproved.yaml, remotion-license-unchecked.yaml, and remotion-assets-thin.yaml.
  • Keep video rendering out of deterministic validators by default; validate manifest evidence, commands, and file presence. Actual render smoke tests belong in the target launch repo.

7. Package, Sync, And Prove

After implementation, run the full existing audit and package checks before syncing the installed skill copy: 

cd skill/b2c-mobile-business-launch
npm run audit
npm run launchbench
npm run test:validators
npm pack --dry-run --json

Then sync ~/.codex/skills/b2c-mobile-business-launch and verify the linked Claude/Agents paths if the user wants the runtime copy updated.

Render Standards

  • Vertical social: 1080x1920, 30fps, 6-20 seconds.
  • Square ad/social: 1080x1080, 30fps or still PNG.
  • Store/app-preview: platform-specific dimensions and truthful app UI.
  • Still checks: render a representative frame with remotion still.
  • Text checks: use Remotion text-measurement guidance for long copy and localization.

QA Standards

  • Real product UI remains visible when the asset claims to show the app.
  • Claims, price, trial, legal, and store language match canonical docs.
  • Captions and silent playback work for social clips.
  • Final assets have render commands, input sources, and output paths.
  • Fallback limitations are recorded when Remotion replaces Higgsfield.

License And Spend Gate

Remotion is lower-cost and often free, but not universally free. The integration should require agents to record eligibility or approval before commercial use when the entity size is unknown. Buying a Remotion company license, using paid rendering infrastructure, or publishing/scheduling the produced assets remains founder-approved.

Failure Modes To Block

  • Agent treats "Higgsfield unavailable" as automatic permission to use Remotion.
  • Remotion output is presented as equivalent to AI-generated campaign creative.
  • Store screenshot or app-preview asset shows mock UI as if it were production.
  • Video/post claims drift from APP_STORE_LISTING.md, REVENUE_OPS.md, or legal docs.
  • Rendered assets lack reproducible source inputs, command proof, or output paths.

Acceptance Criteria

  1. A future agent can decide Higgsfield vs Remotion vs no-production using a clear route matrix and founder-gate policy.
  2. The launch package can include CONTENT_ASSETS.md, content-assets.html, and a manifest that proves how each image or video was produced.
  3. Remotion outputs are tied to DESIGN.md, truthful app UI, owned or licensed assets, and public-claim checks.
  4. LaunchBench catches unapproved Higgsfield fallback, unchecked Remotion license/commercial use, and thin content asset packets.
  5. npm run audit, npm run launchbench, npm run test:validators, and package dry-run pass.

Recommended First Implementation PR

Keep the first PR intentionally bounded:

  1. Add references/remotion-content-assets.md.
  2. Patch route mentions in SKILL.md, paid-tool-routing.md, and tool-recipes.md.
  3. Add CONTENT_ASSETS.md and manifest templates.
  4. Add one validator and one LaunchBench scenario.
  5. Run the existing audit suite and package dry-run.

Defer full Remotion starter scaffolding until the reference and validator contract are agreed. That avoids importing a large toolchain before the skill's routing behavior is locked.
