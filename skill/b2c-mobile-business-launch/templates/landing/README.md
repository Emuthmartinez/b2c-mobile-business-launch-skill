# Landing Section Library

Motionsites-grade landing sections on the project's own brand tokens. This pack is the reusable abstraction of the practice proof in `docs/landing-motion-lab.html`; the doctrine lives in [`references/landing-motion-craft.md`](../../references/landing-motion-craft.md). It is a **web-only** surface: `motion/react` is mandated here and must never be imported by the mobile binary (`check:template-safety` enforces the boundary).

## Drop-in

1. Copy `sections/`, `lib/`, and `motion.css` into a Next.js App Router project (the archetype starters under `templates/app-archetypes/*/starter/` are the expected hosts) or any React SSR site. Astro works via client islands.
2. Install `motion` (the framer-motion successor). Import sections into your page; they are `"use client"` components.
3. Load the promoted brand tokens first: `design-system/tokens.css` (generated by `npm run promote:design-tokens` from `state/theme.tokens.json`), then `motion.css`. Every duration/easing in this pack reads `--motion-*`; there are no magic numbers to retune per launch.
4. Replace the placeholder copy props with launch copy that passes the `check:landing-funnel` copy guardrails.

## The progressive-enhancement contract (enforced, not aspirational)

- **Real text, always.** Copy renders in static HTML. Reveal animations apply only under `html.js` (`motion.css` adds the class gate; `lib/motion-tokens.ts` sets the class on hydration). JS off â†’ the page is a calm, fully legible document.
- **Reduced motion collapses everything.** `motion.css` zeroes durations/transforms under `prefers-reduced-motion: reduce`, and every section also checks `useReducedMotion()` before decorative choreography.
- **Motion never gates LCP.** Sections server-render; choreography hydrates after. No animation may hide above-the-fold text â€” `check:landing-funnel` fails a landing build whose hero text is animation-gated or whose motion ignores reduced motion.
- **Tokens, not magic numbers.** Durations/easings come from `--motion-duration-reveal`, `--motion-duration-cinematic`, `--motion-easing-emphasis`, `--motion-easing-spring`, `--motion-stagger` (plus the base micro-motion scale). Rebrand the launch by editing `state/theme.tokens.json` and re-promoting.

## Baked-video hero (opt-in)

`Hero.tsx` exposes a `videoSrc`/`poster` slot for a Remotion-rendered loop (the Mux-style hero). It is **opt-in per launch** and license-gated: route through [`references/remotion-content-assets.md`](../../references/remotion-content-assets.md) and `TOOL_DECISIONS.md` before rendering. Without the slot the CSS gradient mesh carries the hero â€” lighter and still premium.

## Production notes

- Self-host the real display webfont (subset, `font-display: swap`, preload); the tokens' fallback stack is for dependency-free proofs only.
- A smooth-scroll layer (Lenis) is optional; if added it must degrade cleanly and respect reduced motion.
- Sections are aesthetic-neutral by design: the same components read warm-editorial or dark-glass brands purely from tokens.
